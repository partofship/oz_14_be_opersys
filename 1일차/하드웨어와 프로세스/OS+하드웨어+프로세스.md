컴퓨터 시스템 (연산장치 체계?) = 하드웨어 + 운영체제(base + 종속된 응용 소프트웨어)

### 하드웨어
의미: 물리적 장치 집합

- 중앙처리장치(CPU)
- 메모리(RAM 포함)
- 저장 장치(SSD)
- 입출력 장치(키보드, 모니터, 마우스)

### 운영체제

의미: 하드웨어 자원을 효율적으로 제어·관리 → 사용자 요구사항에 맞게 동작하게끔 만드는 시스템 소프트웨어.

위치: `하드웨어` < `운영체제` > `응용 프로그램`

이런 위치에서 본다면 플랫폼 소프트웨어라고 지칭할 수도 있음

운영 체제는 인터페이스 제공 기능이 필수적. why? 하드웨어 자원(CPU, RAM, 네트워크 등)을 직접 다루는 복잡한 작업을 추상화, 표준화 된 방식으로 응용 프로그램이 자원을 사용할 수 있어야 하기에. 또한 응용 프로그램이 직접 자원에 접근하지 않게 하기 위해서. → 시스템의 안정성, 보안성, 이식성 보장

### 프로세스 관리 · 메모리 관리 · 파일 시스템 관리

- 프로세스 관리: 1) 프로그램(실행 중) → 프로세스(추상화) 2) CPU 사용 순서 결정 3) 관리/책임: 프로세스 생성/종료/상태 전이/동기화/통신

- 메모리 관리: 1) 주소 공간 분리: why? 여러 프로세스가 물리 메모리를 안전하게 공유할 수 있게 2) 가상 메모리: 메모리 사용 효율 maximize 

- 파일 시스템 관리: 사용자 + 응용 프로그램: 일관된 방식으로 데이터 CRUD 할 수 있게 by 보조 저장 장치에 저장된 데이터 → FILE + 폴더(디렉터리) 구조로 조직

### 커널
- 시스템 부팅 시 메모리에서 대기
- OS의 기본+핵심 기능 직접 수행: CPU 제어·메모리 관리·프로세스 스케줄링·장치 드라이버 관리
- 사용자 직접 조작 X
- HOW? GUI, CLI (그래픽유저인터페이스-커맨드라인인터페이스) 통해 간접 명령 전달
- 예시: 바탕화면, 터미널, Shell

### CPU

- ALU(산술논리연산장치) + 제어장치CU + 레지스터 집합Register Set
- ALU: 사칙연산, 논리 연산, 비교 연산 등 실제 계산 작업 담당
- CU: 명령어(from 메모리) 해석, 시스템 전체 동작 순서 제어 (연산 선택, 레지스터와 메모리 선택 및 사용 여부)
- 레지스터 집합: CPU 내 임시 저장소. 연산에 필요한 정보 저장 (데이터, 연산 결과, 현재 실행 위치, 상태 정보)

### 프로그램 실행 과정

- `프로그램` 경로: from 보조 저장 장치 → 주 기억장치 |
- convert into: `프로세스`
- OS: 프로그램을 파일로 인식X `실행 단위` (CPU 자원 소비하는) + 스케줄링 대상 포함 ✅
- 프로세스: 다음을 메모리에 할당받고 독립적 실행 환경 구성: 코드 영역, 데이터 영역, 스택 영역, 힙 영역

### 레지스터

- 초고석 임시 저장 장치
- 저장: 명령어 실행/데이터 처리 필요한 핵심 정보

- CPU 제약: 한 순간에 하나의 명령만 처리.
- ALU: CU가 내린 연산 지시를 수행하는 동안 새로운 명령을 실행할 수 없음
- 레지스터 집합에 저장됨: `대기 상태` 인 중간 데이터 + 다음에 실행할 명령어 정보 (연산에 필요)
- 장점: CPU가 파이프라인 처리/명령여 병렬 처리 → 연산 효율 유지

레지스터: 프로그램 카운터 | 명령 레지스터 | 주소 레지스터 | 데이터 레지스터


- 프로그램 카운터: 저장 - 다음에 실행할 명령어의 메모리 주소
- 명령(Instruction) R.: 저장 - 현재 실행(≒해석)중인 명령어를 메모리에서 가져와서. → CU가 해당 레지스터 내용 해석, 연산 수행 
- 주소(Address) R.: 저장 - CPU가 접근하려는 메모리의 주소
- 버퍼 R.: <b>임시 저장</b> - 메모리-CPU 사이에서 실제 데이터(읽기/쓰기) 데이터의 중간 저장소
- 플래그: 저장 - 연산 결과의 상태. 연산 결과가 0/음수/오버플로우 발생 여부 조건 코드 포함
- 스택 포인터: 현재 스택의 최상단 주소를 가리킴. 제공: 기준 위치 of 함수 호출/복귀/지역 변수 관리
- 데이터 R.: 저장 - 연산에 사용될 실수/정수 값

### 클럭 퍼스
기준 신호 - CPU, 시스템 전체의 모든 동작을 일정한 타이밍으로 동기화하는 기준.

- 1 클럭 펄스 = 11 시간 단위
- 클럭 신호에 맞춰 단계적 진행: 명령어 인출/레지스터 간 데이터 이동/ALU 연산/결과 저장
- ↑ 클럭 주파수 = 초당 처리 가능한 연산 단계↑ → CPU 성능 결정

### 프로그램 실행 과정

명령어 사이클: FETCH, DECODE, EXECUTE

이 세 단계는 프로그램이 종료될 떄까지 클럭 펄스마다 반복

- [메모리] 저장 명령어 → CPU 전달 → 처리
- Fetch: 프로그램 카운터가 가리키는 주소에 위치한 명령어를 메모리에서 CPU로 가져오며, 가져온 명령어는 인스트럭션 레지스터에 저장. 동시에 프로그램 카운터는 다음에 실행할 명령어의 주소로 증가
- Decode: 제어장치가 인스트럭션 레지스터에 저장된 명령어를 해석하여 어떤 연산을 수행해야 하는지, 어떤 레지스터나 메모리 주소를 사용하는지 분석
- Execute:해석된 명령에 따라 ALU, 레지스터, 메모리 등이 실제 연산과 데이터 이동을 수행하며, 결과는 레지스터 또는 메모리에 저장되고 플래그 레지스터도 함께 갱신


### 인터럽트

매커니즘: 명령어 실행 흐름을 일시적 중단 → 우선 처리해야 할 사건 발생을 CPU에 알림

이 구조로 CPU는 입출력 장치 동작을 계속 감시할 필요 X

전체 시스템 처리 효율, 응답성 향상

- 비동기 인터럽트: 입출력 장치에서 발생 = CPU와 무관한 외부에서 발생 (키보드, 디스크, 네트워크 신호 등)
- CPU: 인터럽트 발생 시 실행 중이던 명령어 상태 저장
- 제어 이동: 해당 인터럽트에 대응하는 인터럽트 서비스 루틴
- 인터럽트 서비스 루틴? 미리 정의된 처리 프로그램 for 특정 인터럽트 신호. (기능 수행: 장치 상태 확인/데이터 전송/오류 처리)
- 처리 완료 → CPU: 저장된 원 실행 위치 복구/기존 프로그램 재개

### 프로세스 제어 블록 PCB, 컨텍스트 스위칭
`Process Control Block`
- OS는 프로세스를 매우 빠른 속도로 번갈아 실행
- Issue: 실행 상태를 정확히 저장/복원해야 멀티테스킹이 가능
- 프로세스를 식별하고 제어하기 위한 핵심 관리 자료구조로써 모든 정보가 저장된 것: 한 프로세스마다 하나의 프로세스 제어 블록
- `컨텍스트 스위칭`: CPU가 서로 다른 프로세스를 일정 시간이 지날때마다 중단/바꿔서 실행, 초당 수천~수백만 번 반복.

- PCB 저장되는 정보: 프로세스 식별 정보 | 프로세스 상태 | CPU 레지스터 정보 | 스케줄링 정보 | 메모리 관리 정보 | 입출력, 자원 정보
    - 프로세스 식별 정보
    - 프로세스 상태
    - CPU 레지스터 정보
    - 스케줄링 정보
    - 메모리 관리 정보
    - 입출력, 자원 정보
